**** Concepts
***** Var/log/message
- A centralized record of system activity
- Aims at storing valuable, non-debug and non-critical messages. This log should be considered the "general system activity" log.
- Typically will contain:
  - System boot and shutdown events
  - Hardware activity
  - Kernel messages
  - Network events
  - Background services (daemon) activity
  - Errors and warnings from various subsystems
***** Module
- A system that allows users to dynamically modify their shell environment to load specific software.
- The 'module' command is a user interface for managing software environments.
- It works by reading modulefiles, which define how an application should be set up.
- Modulefiles configure things like:
  - PATH
  - LD_LIBRARY_PATH
  - Environment variables
  - Users can load/unload software on demand without permanently changing their system.
***** INODE
- A data structure which describes a filesystem object such as a file or directory. It stores all metadata about a file excepts its name and actual content.
- Each inode stores the attributes and disk block location(s) of the object's data.
- Directores are lists of names assigned to inodes.
- A directory contains an entry for itself, its parent, and each of its children.
***** Hard Link
- Merely an additional name for an existing file on Linux or Unix-like operating systems.
- Any number of hard links, and thus any number of names, can be created for any file.
- Hard links can also be created to other hard links, however they cannot be created for directories, and cannot cross filesystem boundaries or span across partitions.
  - ln file_name hardlink: creates a hard link to an existing file.
  - ls -i: lists all files with inodes associated with each file
***** Soft Link
- A special kind of file that points to another file
- A symbolic link does not contain the data in the target file, it simply points to another entry somewhere in the file system.
- Has the ability to link to directories, or to files on remote computers networked through NFS. Also, when you delete a target file, symbolic links to that file become unuseable, whereas hard links preserve the contents of the file.
  - lns -s file_name1 soft_link1: creates a soft link between one file and another.
***** Filesystems
- A file system is the method and structure an operating system uses to organize, store, and manage data on a storage device (like a hard drive or SSD).
****** FAT32
- Stands For: "File Allocation Table 32"
- An older, widely compatible file system commonly used on removable and small storage devices.
- Larger external hard drives of ~1TB will likely come formatted with NTFS.
- You'll only want to use FAT32 with small storage devices or for comptability with other devices like digital cameras, game consoles, set-top boxes, and other devices that just support FAT32 and not the newer NTFS file system.
- Characteristics are:
  - 4 GB Max file size
  - Less secure
  - No permissions or journaling
****** NTFS
- Stands For: "New Technology File System"
- The default file system for modern Windows Systems (Since XP)
- Supports:
    - Larger File sizes
    - File Permissions ([[id:f5e195a7-e325-49f3-aa9c-1118fadd759f][ACLs]])
    - Journaling
    - Compression
    - Encryption
****** HFS+
- Stands For: "Hiearchical File System Plus"
- A file system historically used by Apple for macOS systems before newer formats replaced it.
- Designed for macOS environments
- Supports:
  - Large files
  - Metadata (resource forks, timestamps, etc.)
  - Was the default Mac file system before 2017
****** APFS
- Stands For: "Apple File System"
- The modern file system used by Apple for macOS, iOS, etc.
- Optimized for SSD Storage.
- Supports:
  - Snapshots (point-in-time backups)
  - Strong encryption (multi-key encryption)
  - Fast File operations (copy-on-write)
  - Space sharing between volumes
****** Ext2/Ext3/Ext4
- Stands for: "Extended File System"
- Most commonly used on Linux.
- Ext2 is an older file system -> lacks important features like journaling-if the power goes out or a PC crashes while writing to an ext2 drive, data will be lost.
- Ext3 adds these robust features at the cost of speed.
- Ext4 is the most modern & fastest -> it's the default file system on most Linux distros now.
- Windows and Mac do not support Ext filesystems, however they can with third-party tools.
- For this reason -> it's often ideal to format your linux partitions with Ext4 and leave removable devices formatted with FAT32 or NTFS.
****** Btrfs
- Stands For: "Better File System"
- A modern Linux file system designed for advanced features, scalability, and data integrity.
- Will not be set by default, but will likely replace it eventually.
- Designed for large-scale storage systems
- Focuses on:
  - Data integrity
  - Flexibility
  - Advanced volume management
******* Core Features
- Copy-on-Write (CoW)
  - Writes new data instead of overwriting existing data
  - Improves readability and enables other features
- Snapshots
  - Create instant, space-efficient backups of a filesystem state
  - Useful for:
    - System rollback
    - Backup strategies
- Built-in RAID & Volume Management
  - Can manage multiple disks without needing LVM
  - Supports RAID-like configurations natively
- Checksums (Data Integrity)
  - Detects data corruption automatically
  - Can self-heal (with redundancy configured)
- Subvolumes
  - Acts like separate filesystem sections within one partition
  - Useful for organizing data or isolating workloads
***** Raid
- Redundant Array of Independent Drives
  - Takes multiple hard drives and puts them together through a controller to provide a single layer capacity drive.
  - RAID storage may be software or hardware based, and can be used for three distinct purposes:
    1) Capacity
       -> 2 hard drives being linked together as a single drive to the operating system. Making a virtual drive that is twice the capacity
    2) Security
       -> Using some space on the drives to clone the data that is written to both drives. Once again, with two drives we can make it so that data is written to both drives. If one fails, the other one is redundant.
    3) Performance
       -> In the same example of two drives being used as a single drive, the controller can split a data chunk into two parts and then put each of those parts on a separate drive. This effectively doubles the performance of writing or reading the data on the storage system.
****** Raid Levels
1) Raid 0 (Striping)
   - Lowest level of RAID
   - No redundancy
   - Takes two or more drives and combines them to make a larger capacity drive.
   - Data blocks are broken up into data chunks and then written in order across the drives.
   -> This process is called 'striping'
   - Data can be wrtten simultaneously to the drives, effectively multiplying the speed of the drives.
 - Pros:
   - Very fast (parallel read/write)
   - Full capacity utilization
 - Cons:
   - No fault tolerance
   - One drive failure = total data loss
2) Raid 1 (Mirroring)
   - Full level of redundancy
   - Uses 2 or more drives
   - All data that is written to the system is copied to each drive in a level 1 array.
     -> This process is called "mirroring"
 - Pros:
   - High reliability
   - Simple recovery
 - Cons:
   - 50% usable capacity (2 drives -> 1 usable drive)
3) Raid 1+0 or 10
   - A combination of both RAID levels 1 and 0.
   - Minimum of 4 drives
   - Drives are paired into mirrored sets
     - Then data is striped across those mirrored pairs
 - Pros:
   - High performance
   - High redundancy
   - Fast rebuild times
 - Cons:
   - Requires more drives
   - 50% capacity loss
4) Raid 5
    - Minimum of 3 drives
    - Uses striping + distributed parity
    - Data is split across drives (striping)
    - Parity information is distributed across all drives
    - Parity: calculated data
    - Allows the system to reconstruct missing data if a drive fails
13) Pros:
    - Capacity
    - Performance
    - Fault Tolerance
14) Cons:
    - Slower writes (parity calculation)
    - Only tolerates 1 drive failure
***** Zombie Processes
- Zombie processes are processes that have completed its execution but still has an entry in the system's process table.
- You can view zombie processes using htop, btop, or top
- You can also run: ps aux | grep Z (aux prints every process on the system using BSD syntax. A process with
a ‘status’ of ‘Z’ is a zombie process.)

***** Cron
- A daemon commonly used in linux that runs processes at regular intervals
- It checks particular folders on your system for scripts to run in a series of folders including /etc/cron.hourly, /etc/cron.daily, /etc/cron.weeky, and /etc/cron.monthly. There is also a file called /etc/crontab.
****** Each line of a crontab file
****** # m h dom mon dow - command
- The minute of the hour the command is to run (m)
- The hour of the day the command is to run (h)
- The day of the month the command is to run (dom)
- The month the command is to run (mon)
- The day of the week command is to run (dow)

  -> 30 18 * * * tar -zcf /var/backups/home.tgz /home/

***** Pipe
- Pipes allow you to chain commands together so one command’s output becomes another’s input.
- Ex. # command1 | command2
  - command1 -> produces output
  - command2 -> takes that output as input and processes it
- Ex. # dmesg | less
  - dmesg -> prints system boot/kernel messages
  - less -> lets you scroll through output interactively
- This solves the problem of output being too fast to read.
****** Common Uses
- ps aux | grep ssh
  -> Filters output
- ls | sort
  -> Sorts
- cat file.txt | wc -l
  -> Counting

***** sed
- The primary use of Linux command sed, which is short for "stream editor" is to modify each line of a file or string by replacing specified parts of the line.
- It makes basic text changes to a file or input from a pipeline
  - Ex. songs.txt contains these lines:
    1) Justin Timberlake, Title 545, Price $6.30
    2) Johnny Cash, Title 482, Price $6.50
    3) Elvis Presley, Title 335, Price $6.50
    4) John Lennon, Title 271, Price $7.90
- If we want to change all price occurrences of $6.30 to $7.30. To do so, we can use the sed command in this way:
# sed 's/6.30/7.30/' songs.txt > songs2.txt
- This syntax makes the change and saves the modified file as songs2.txt. The new file contains:
  1) Justin TImberlake, Title 545, Price $7.30
  2) Johnny Cash, Title 482, Price $6.50
  3) Elvis Presley, Title 335, Price $7.30
  4) John Lennon, Title 271, Price $7.90
- Sed is also frequently used to filter lines in a file or stream. For example, If you only want to see the lines containing "John", you would use the following sed command:
  # sed -n '/John/p' songs.txt > johns.txt
  - The new johns.txt file will contain the following lines:
    3) Johnny Cash, Title 482, Price $6.50
    4) John Lennon, Title 271, Price $7.90

*** OS Boot Proces
**** Power
**** BIOS
- Stands for: Basic Input/Output System
- Searches, loads, and executes the bootloader program. Once the boot loader program is detected and loaded into memory -> BIOS gives the control to it. (TLDR: BIOS loads and executes the MBR bootloader.)
**** Post
- Stands for: Power on Shelf Test
- This test checks all connected hardware, including RAM and secondary storage devices to ensure that they're functioning.
**** MBR
- Stands for: Master Boot Record
- Located in the first sector of the bootable disk. It points to the boot record of the selected operating system.
- TLDR: MBR loads and executes the bootloader.
**** GRUB
- Stands for: Grand Unified Bootleader
- GRUB is a bootloader, and it has knowledge of the filesystem.
- GRUB depends on its configuration file, which specifies things like how long the timer until it chooses the default kernal image, what kernel images to list, the GUI, etc.
**** Kernel
- Mounts the root file system as specified in the "root=" grub.conf
- Kernel executes the /sbin/init program
- The kernels are located in the /boot directory, along with an initial RAM disk image (initrd).
***** initrd
initrd is used by the kernel as temporary root file system until kernel is booted and the real root file system is mounted. It also contains necessary drivers compiled inside, which helps it to access the hard drive partitions and other hardware.
**** OS (systemd)
- uses its configuration file -> /etc/systemd/system/default.target, to determine which state or target, into which it should boot the host.
- For a server, the default is more than likely to be the multi-user.target which is runlevel 3.
- If graphical.target (runlevel 5) is the default, than you should see a GUI login.
*** BIOS vs CMOS
- BIOS is a small program that controls the computer from the time it powers on until the OS takes over. The BIOS is firmware, and thus cannot store variable data.
- CMOS (Complementary metal oxide semiconductor) is a type of memory technology, but generally refers to the chip that stores variable data for startup. A computer's BIOS will initialize and control components like the hard drive controllers and the computer's hardware clock, but the specific parameters for startup and initializing components are stored in the CMOS.
*** Comparing MBR to GPT
- Prerequisite: You have to partition a disk before you can use it.
- Master Boot Record (MBR) and GUID Partition Table (GUID) are two different ways of storing the partitioning information on a drive.
- This information includes where partitions start & begin, so your operating system knows which sectors belong to each partition and which partition is bootable. This is why you have to choose MBR or GPT before creating partitions on a drive.
*** MBR
- Stands for: Master Boot Record
- First sector of a hard disk
- Holds the information on how the logical partitions, containing file systems, are organized on that medium
- The MBR also contains executable code to function as a loader for the installed operating system-usually by passing control over to the loader's second stage, or in conjuction with each partition's volume boot record (VBR).
- This MBR code is usually reffered to as a boot loader.
**** Associated Terms
- Sector: The smallest unit that can be accessed on a hard disk
- Cluster: A group of sectors that make up the smallest unit of disk allocation for a file within a file system. (TLDR: A cluster size is the smallest amount of space a file can take up on a computer. A common size is 512 bytes. A common cluster size is 8 sectors.)
*** GPT
- Stands for: GUID Partition Table
- A new standard that's gradually replacing MBR.
- It's associated with UEFI, which replaces the old BIOS with something more modern. GPT, in turn, replaces the old MBR partitioning system with something more modern.
- It's called GUID Partition Table because every partition on your drive has a globally unique identifier, a random string so long that every GPT partition on earth likely has it's own unique ID.
**** Quick Facts
- GPT doesn't suffer from MBR's limits (2TB Drive limit).
- on an MBR disk, the partitioning and boot data is stored in one place. This means if data is overwritten or corrupted, you're in trouble. In contrast -> GPT stores multiple copies of this data across the disk.
- GPT stores CRC check values to check that its data is intact.
- MBR did not do any of this shi.

Q: Can you explain the OSI model?
A: - OSI stands for open system interconnection
   - It is a reference model that was developed in the early 1980s and published in 1984 by organization ISO
   - Its purpose is to create a vendor-neutral and universal framework for how different networking systems should communicate and encapsulate data
   - Before the OSI model certain vendors could only use their own proprietary methods of communicate

Q: How would you diagnose a machine? / Tell me about SSH
A: “One of the first things I’d check when diagnosing a machine is whether it’s reachable over SSH. If I can successfully SSH into the system, that immediately tells me several layers are likely functioning correctly — the machine is powered on, reachable on the network, DNS or IP routing is likely working, the SSH service is running, and the OS is responsive enough to accept remote sessions.

If SSH fails, then I can start narrowing the problem space. For example:

If the hostname doesn’t resolve, that points toward DNS issues.
If there’s no route or timeout, I’d investigate network connectivity, VLANs, ACLs, routing, or physical connectivity.
If the port is closed or refused, the host may be up but the SSH daemon may not be running.
If authentication fails, then connectivity exists but there may be credential or access-control issues.

SSH is useful because it quickly helps distinguish between network-level problems, service-level problems, and system-level problems.”

Q: What happens when you type a url into a web browser?
A: The system then checks several places to resolve the domain name into an IP address — such as the browser cache, OS cache, hosts file, and finally DNS servers if needed.

Once DNS resolution succeeds, the client establishes a connection to the server IP. For HTTPS, this usually means:

a TCP three-way handshake,
followed by a TLS handshake to negotiate encryption and validate certificates.

After the secure session is established, the browser sends an HTTP request — like a GET request — containing headers, cookies, and other metadata.

The request traverses routers and network infrastructure until it reaches the destination server or load balancer. The server processes the request, possibly querying databases or backend services, then returns an HTTP response with status codes, headers, and content.

The browser receives the response, parses the HTML, builds the DOM and render tree, downloads additional resources like CSS and JavaScript, executes scripts, and finally renders the page visually to the user.

Throughout the process, there are additional optimizations happening like caching, CDNs, connection reuse, compression, and asynchronous resource loading.”

** Googlyness
*** Article by Jeff Sipe
Let’s start by focusing on the motto — Do the right thing. Google strives to hire people who are honest and kind, so the biggest item/concept to focus on, is being a good person. This shows up during the interview in a couple of ways. The first item is based on interactions, not only the way you interact with the interviewers, but the way you interact with the receptionist, the person serving you a coffee or lunch in the cafe, the people you pass in the hall, etc. Make sure all of your interactions are courteous, friendly, and positive. Second is your interview answers, these answers should demonstrate how you have done the right thing throughout your career.

A critical aspect of interviewing is the use of words, let’s explore a few items:

Positivity — I personally encountered something completely new when I worked at Google, people who would only respond to positivity. Anything negative, words, pitch, or tone would shut them down, literally anything. And while you need to be positive in all interviews, when interviewing at Google and thinking about Googleyness, take it to the next level. And by next level I want you to focus on turning the negative, positive. For example, change negative words like failure, conflict, and weaknesses by rephrasing these items as an opportunity. Because in all instances when something does not go according to plan it is our opportunity to learn from it and apply that learning in the future.

Gender — All workforces are made of he’s, she’s, and people that identify as gender neutral or non-binary. So it is important to not constantly refer to an Engineer, Salesperson, or CEO as he. It will instantly communicate gender bias and downtick your success rate, switch it up. It shows that when you are working with others you are always considering inclusivity.

Political Correctness — I know this term is a little dated, but do not skate the line on this item, it likely will not end in a positive result. Do not take chances with controversial topics because it will strongly impact how you are viewed from a Googliness perspective. Google is an incredibly liberal company where everyone is accepted, this is the environment, and they will be looking for that match during the interview.

Well, it wouldn’t be an article about Googleyness without mentioning the famous Lazlo Bock, Google’s most well known Human Resources employee. Of all the articles written on Googliness (besides this one of course), Lazlo’s is the best at truly defining this concept. Lazlo highlights five important concepts in his definition of Googleyness

Attributes like enjoying fun (who doesn’t) — Google employees work hard, but they also are playful and like to have a lot of fun, from going to cafe and grabbing lunch with a friend, to taking a break to shoot a quick game of pool with a colleague. I used to blast Rebecca Black early on Friday mornings to get people energized and pumped up for the day for example. During the interview, be able to provide these anecdotes/examples about having fun and overall implement a little bit of fun into your answers.

A certain dose of intellectual humility (it’s hard to learn if you can’t admit that you might be wrong) -

Having intellectual humility will show up in your interviews in the form of being humble, admitting you had an area/s of growth, needing others to take the lead, needing advice/guidance from others, and trusting that others are acting in your best interests. This last item is critical, if we operate from a place of thinking positivity about what others are doing, it will make the way we communicate with them that much stronger. Lastly, a constant focus throughout the interview on learnings and the application of those learnings. Intellectual humility is about your ability to learn and grow, talking about it will greatly increase your chances of success.

A strong measure of conscientiousness (we want owners, not employees) — Have high standards and NEVER make an excuse in an interview. Own everything and talk about how you improved. Focus on how you worked really hard to get better in that specific area through continuing education, how you worked to improve your listening skills, whatever it might be.

Comfort with ambiguity (we don’t know how our business will evolve, and navigating Google internally requires dealing with a lot of ambiguity) — This specific concept is called out by both Lazlo and Google on their carers page. Comfort with ambiguity is all about soft skills, like questioning, listening, empathy, collaboration, consideration, etc.

Evidence that you’ve taken some courageous or interesting paths in your life — Part of creating an inclusive environment is recognizing the value of people coming from different backgrounds, people that have taken different career paths, etc. This could be about taking a non-traditional path, this could be about leaving the cushy job to run a startup, or switching focus/areas skills in the middle of your career. Whatever it might be, focus on how that path has helped you thrive and will help you provide a unique/interesting lens if given the opportunity to work at Google.

Go to the source, and what better source for figuring out this Googliness thing than Google. Remember, do not rely on third party sources, they often aggregate data that is completely false/incorrect. Trust the information coming directly from the organization and those that have been in the arena, those that have actually worked for or with the company. Google defines Googleyness as four items:

How you work individually and on a team — This shows up in focusing on what you did, but always recognizing and giving praise to those that supported you/were critical to your success. You always want to highlight the strength/s of your team, whether these were collaborators, stakeholders, vendors, etc. And don’t forget to talk about what you learned by working with others, how they helped you grow/develop.

How you help others — Whether you are interviewing for an individual contributor role or people management position, caring and helping others is a core function of Googleyness. This could be business or personal related. People experience life events during their career, so when you talk through your examples of helping others, talk about how you listen, show empathy, have compassion, and what you personally have learned by helping others.

How you navigate ambiguity — Googliness is about appreciating and finding joy in the unknown, welcome to Google! The interviews will challenge your ability to problem solve with extremely limited information. It is important that you respond to questions that explore this unknown with positivity and that you have a little fun too. How to overcome some of the ambiguity in the interviews is to make it conversational and ask clarifying questions. Build in pauses and space to interact with the interviewer, and check in with them often during your solution.

How you push yourself to grow outside of your comfort zone — Most often this shows up as taking on projects outside of your core function, mentoring and leading others, having challenging conversations, and pushing yourself when it is uncomfortable. Have great examples of times when you have done these items, prepared for your interview. This could also be about growth related to the role outside of work, such as you took a Python class to get better at your job or you read a book on leadership to become a better people manager, etc. And lastly, are you demonstrating growth during the interview. I would look for this when I interviewed potential Googlers, could this person evolve and get better during our 45 minute conversation.

I want to end by briefly touching on the power of emotional intelligence, defined as the ability to understand, use, and manage your own emotions in positive ways to relieve stress, communicate effectively, empathize with others, overcome challenges, and defuse conflict. Think about this concept/definition and how it directly correlates to all the other items discussed in this article.

If you want to get hired by Google, you will have to demonstrate Googliness throughout all your interactions and you will be tested on it in the interview process. Good luck and I hope this article helps!
